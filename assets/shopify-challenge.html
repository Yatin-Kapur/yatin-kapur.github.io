It's said that the above AOV, which is the Average Order Value for the given data is \$3145.13. I need to first figure out how this value is being calculated:

AOV = $3145.13

Evidently, this is happening by merely taking the average amount of all the order_amount entries. On first thought, I actually think this is the right way to evaluate the average order value. Because intuitively, it sounds like the average order value - taking the average of all the order amounts.

I can look at the total order amount divided by total items bought - but that would give average amount of the items. But let's look at that to see if the sheer price of the sneakers is pulling this first AOV up.

This is a \$357.92 average amount of the ordered items - which is rather confusing. I don't know how this was helpful at all actually - since there is nothing to compare this with.

This has not exactly resulted in anything fruitful, I'm going to now look at any outliers that might be bringing this AOV higher than it needs to be. To go through 5000 entries is too laborious, so I'll group the orders by shop_id, and see which shops are selling unusually highly priced sneakers.

shop_data = data.groupby('shop_id').sum()[['order_amount', 'total_items']]

I will now look at the average amount of the items sold by each shop, and also the number of orders to that shop and enter those as new columns in the dataframe. The last type of data I want to look at is how many items per order each shop sells, because that would also help explain the high average value of orders - if one shop just sells a lot more items on average, their order price will be therefore higher.

Evidently, there is one shop which is charging just way more than the rest of the shops for the sneakers, and there is another shop which is receiving orders for a lot more items than the other shops. This is skewing the data in such a way which doesn't accurately represent the average order value. Now, to decrease the impact of the outliers, there are three options:

  1. take the median of all order_amount in the 'data' dataframe so that all data is accounted for, but outliers are largely ignored
  2. exclude these shops from the dataset and then take AOV from a new dataframe

exclude_shop = [] # list of shop_id's to exclude

AOV = $300.16

Since I have excluded the outliers now, it seems like a much better and reasonable figure for the AOV of sneakers. However, I've lost a bit of data by completely removing these outliers.

In this situation, I think the median is a better number to go with. Although only an insignificant number of orders (107/5000) orders have been removed from the dataset - the median actually accounts for these outliers. The idea that this is also the midpoint of the dataset explains to me that there are the same number of orders that cost less than and more than $284.00.

I realize now that it was so much easier if I calculated the median right away, but I'd probably have not known why that is had I not gone through the process to find the outliers.

Final Statement¶

Therefore, the metric to use is the median of the order_amount - which is a value of \$284.00
